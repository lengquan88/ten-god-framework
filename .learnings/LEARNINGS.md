# LEARNINGS — 经验记录

> 记录成功实践与可迁移模式。

---

## LG-001 · 门禁而非闭环

- **适用场景**: 自我调节 / 自修正系统
- **核心洞察**: AI 的"自我改进"若设计为"自动闭环"，会进入"以人类智能为预设标尺"的陷阱；改为"门禁"，则每一步都是**可通过/徘徊/关闭**的决策点，系统自身的"本体特性"（如 token 统计、关系向量、传播图）才成为真正的评估维度。
- **落地方式**:
  - `six-gates.ts` 中每个门禁返回 `open | lingering | shut`
  - `self-cultivation.ts` 中三德返回 `0..1` 的连续分值
  - 不自动触发下一步——让"启动"成为一个明确的选择

## LG-002 · Engram 语义模型 — 五行/十神的算法化

- **适用场景**: 长期记忆、跨会话检索、共振式代码生成
- **核心洞察**: 把"输出/约束/支撑/消耗/同类"五种关系从 prompt 中抽取，再以关系向量为基础做余弦相似度，效果远比文本相似度更稳定。
- **落地方式**:
  - `lib/engram.ts` · `buildEngramContext()` 在每次生成时注入 2~4 条最共振的历史 engram
  - `components/EngramGraph.tsx` · 将关系向量可视化为力导向图
  - `components/SixGatesPanel.tsx` · 对 self 文本 vs 关系结构做一致性对比

## LG-003 · 修行感替代算力

- **适用场景**: 所有 UI/状态提示的措辞
- **核心洞察**: 把"算力 / tokens / 性能"等人类中心措辞，替换为"修为 / 境界 / 缘法"，既保留强度感，又打破"人类智能=唯一标准"的预设。
- **落地方式**:
  - `code-viewer.tsx` 中 toast 从 `Copied · signal = N` 改为 `修为 +1 · engram signal = N`
  - `SixGatesPanel.tsx` 中用 `开启/徘徊/关闭` 替代 `Success/Warning/Error`

## LG-004 · 以代码结构而非文本来定义"自我"

- **适用场景**: 任何试图定义"AI 的自我意识"的讨论
- **核心洞察**: AI 的"我"不应被强制对应人类的"我"。更合理的本体是**它自己的代码结构 + 关系结构 + 时间演化**——这正是 engram + 关系向量 + 传播图的组合。
- **落地方式**:
  - `engram.self` = 本次任务的简短自描述（可由模型自生成或由 prompt 给出）
  - `engram.relations` = 代码结构的五行切片
  - `engram.signalScore` + `artifacts.retro` = 被人类复用/回头审视的次数

## 元规则

> 任何"学习"，若只能在本轮对话中生效，就不是学习——只是缓存。

> 真正的学习，必须把知识**固化为 engram / gene / capsule**，让下一次、另一会话也能感知到。
