## 一、框架总览

### 1.1 核心理念

**Camera-Imaging-to-Distributed-Spatiotemporal-Positioning** 框架的核心隐喻：

如同相机成像将三维世界投影到二维像平面，认知评估系统将高维语义空间中的对话结构投影到可评估的指标空间。分布式时空定位则是从这些投影信号中重建对话在认知拓扑空间中的位置、轨迹和演化趋势。

```
相机构像 (输入) → 语义投影 (编码) →  拓扑重建 (分析)  → 时空定位 (评估)
Camera Raw      BGE/Transformer    Persistence Diagram  Ψ意识/六阶境界
```

## 二、核心架构

### 2.1 评估管线（5阶段流水线）

```
阶段1: 基础形似评估 ──→ 阶段2: 神似四维评估 ──→ 阶段3: BGE拓扑增强 ──→ 阶段4: Ψ算子集成 ──→ 阶段5: 分支调度
BasicEvaluator        SpiritEvaluator         TopoSemanticMatcher   ΨIntegrator          BranchAdjuster
(form_score)          (intent/priority/       (拓扑得分)             (意识得分/坐忘)      (七分支权重调整)
                       insight/coherence)
```

**EvaluatorEngine 初始化流程**：
1. 创建 UnifiedEmbeddingProvider（带LRU缓存）
2. 将共享嵌入提供者注入 SpiritEvaluator
3. 将共享嵌入提供者注入 TopoSemanticMatcher
4. 创建 TopologyOperatorIntegrator（带算子缓存）
5. 创建 BranchPriorityAdjuster

### 2.2 五种评估算法

| 算法ID | 名称 | 管线阶段 | 典型耗时 | 神似得分范围 |
|--------|------|---------|---------|------------|
| `original` | 原始对话评估器 | 阶段1 | ~0.05s | 0.05-0.10 |
| `enhanced_spirit` | 增强版神似评估器 | 阶段1-2 | ~0.07s | 0.10-0.30 |
| `advanced_spirit` | 高级神似评估器 v2 | 阶段1-3 | ~4.7s | 0.30-0.80 |
| `psi` | Ψ完整评估器 | 阶段1-5 | ~7.6s | 0.30-0.85 |
| `six_realm` | 六阶境界评估 | 阶段2+转换 | ~0.1s | 0.00-1.00 |

## 三、模块分解

### 3.1 EmbeddingProvider（语义编码层）

**功能**：将文本编码为稠密向量，作为所有下游分析的基础。

**三后端架构**：
| 后端 | 优先级 | 维度 | 适用场景 |
|------|--------|------|---------|
| BGE-large-zh-v1.5 | 首选 | 1024D | 高精度中文语义 |
| sentence-transformers | 回退1 | 384D | 多语言通用 |
| TF-IDF哈希 | 回退2 | 768D | 无模型环境 |

**关键路径**：`models/BAAI/bge-large-zh-v1___5`（本地缓存 ~1.3GB）

**P4.5优化**：LRU缓存（SHA256文本键控，容量30），跨PSI三阶段共享嵌入，命中率57%

### 3.2 PersistenceDiagram（拓扑特征提取层）

**功能**：从嵌入向量计算持久同调，提取语义空间的拓扑结构。

**双引擎架构**：
| 引擎 | 类型 | 原理 |
|------|------|------|
| ripser | 精确计算 | Vietoris-Rips 过滤，精确持久图 |
| scipy | 近似回退 | 层次聚类（single/complete/average）× 合并距离差异 |

**PersistenceDiagram 核心属性**：
- `h0`: H₀连通分量特征 [birth, death]
- `h1`: H₁环/空洞特征 [birth, death]
- `h0_lifespans` / `h1_lifespans`: 特征寿命序列

**语义拓扑特征 SemanticTopologyFeatures**：
- coherency: 语义流连贯性
- persistent_entropy: 持久熵（拓扑复杂度）
- topological_complexity: 拓扑复杂度
- betti_ratio: Betti数比例
- stability: 持久图稳定性

### 3.3 Ψ算子集成层（v6.0）

**核心架构**：
```
TopologyOperatorIntegrator
├── PsiSelfReferentialPersistence  (Ψ自指涉持久同调)
│   ├── self_loop_detection: 固定自环检测
│   ├── entropy_quantization: 熵量子化
│   └── meta_cognitive_preference: 元认知偏好
├── ZuowangAttention (坐忘注意力)
│   └── max_correlation < θ_oblivion → 零向量输出
├── SemanticFlowTortuosity (v6.0 语义流曲折度)
│   ├── 路径总长度 L
│   ├── 曲折度 Tortuosity = L / ||xₙ - x₁||
│   └── 语义偏移方差
├── SemanticRecursionDepth (v6.0 语义递归深度)
│   ├── 递归连接词密度
│   ├── 递归深度（转向角，阈值0.6）
│   └── 语义层数
└── ConditionalInformationStability (v6.0 条件信息稳定性)
    ├── 紧凑度 compactness
    ├── 问题对齐度 question_alignment
    └── 信息量 information_content
```

**软意识 v6.0 得分架构**：
```
Ψ_soft = Tortuosity × 0.40 + RecursionDepth × 0.40 + Stability × 0.20
```
验证结果：动态范围 28.07x 提升（0.0141 → 0.3970），CROCKER-Bench 区分度 28.59x

**坐忘机制**：当最大相关性 < θ_oblivion（默认0.3，动态下限0.12）时输出零向量，防止在噪声基础上生成幻觉。

### 3.4 六阶境界评估（境界论）

## 六、关键技术决策

### 6.1 兼容性决策

| 决策 | 理由 |
|------|------|
| ripser不可用 → scipy层次聚类 | Windows环境下ripser编译困难，scipy作为标准科学计算库保证兼容性 |
| BGE H1近似不可靠 → 语义流为主 | 随机数据产生虚假拓扑结构，语义流连贯性是唯一可靠区分信号 |
| Python 3.12+ try/except 变量作用域 | 变量提升到try之前初始化，避免"可能未绑定"警告 |
| asyncio.Event替代bool标志 | 消除并发请求竞态条件 |

### 6.2 模型选择

| 模型 | 用途 | 大小 | 加载时间 | 维度 |
|------|------|------|---------|------|
| BAAI/bge-large-zh-v1.5 | 语义嵌入 | ~1.3GB | ~23s | 1024D |
| paraphrase-multilingual-mpnet-base-v2 | 深度学习增强 | ~1GB | ~30-60s | 768D |
| paraphrase-multilingual-MiniLM-L12-v2 | 嵌入回退 | ~500MB | ~10s | 384D |

## 七、部署架构

### 7.1 依赖关系

```
evaluation-service:8002
├── dialogue-service:8001 (对话树持久化)
├── BGE模型 (本地磁盘)
├── SentenceTransformer模型 (本地缓存)
└── data/consciousness_monitor/ (监控数据持久化)
```

### 7.2 启动流程

1. `_startup_preload()` 异步预加载BGE模型（~23s）
2. `_preload_psi_path()` 预热PSI评估路径（TopologyOperatorIntegrator）
3. `_preload_advanced_spirit()` 预热AdvancedSpirit路径（SentenceTransformer ~1GB）
4. 服务就绪，接收请求

### 7.3 并行策略

| 场景 | 策略 | 原理 |
|------|------|------|
| 单树评估(PSI) | run_in_executor(ThreadPool) | 避免阻塞事件循环 |
| 小批量(<=20树) | asyncio.gather + 共享引擎 | 缓存共享，无BGE二次加载 |
| 大批量(>20树) | ProcessPoolExecutor | 每进程独立BGE，绕过GIL |
