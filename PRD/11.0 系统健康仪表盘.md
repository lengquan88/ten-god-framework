## 一、当前缺失的可观测性缺口

当前所有面板都展示"系统的输出"（意识得分、推背图、认知地图），但没有一个面板展示"系统本身的健康状态"：

| 维度 | 覆盖 | 缺失 |
|------|:---:|------|
| 输出质量 | ✅ 评估+推背图+画像 | — |
| 模块完整性 | ✅ test_suite 33项 | — |
| 边界鲁棒性 | ✅ 天启测试40项 | — |
| **运行时健康** | ❌ — | 节点质量信誉、漂移警告、延迟、吞吐 |
| **趋势追踪** | ❌ — | 长期退化、版本回归、模型对比 |

六论诊断：
- 本体论缺口 — 系统存在但无法观测自身（卷56「元气论」：气不显于外则无法调摄）
- 元认知论缺口 — 系统无自我审计闭环（卷94「坐忘论·真观」：不能观己则无法得道）

### 七分支在仪表盘中的角色

| 七分支 | 仪表盘指标 | 表达式 |
|--------|----------|-------|
| 可见性 V(x) | 节点质量信誉面板 | V = f(access_ctrl, health_score) |
| 时效性 T(x) | 庚申周期检查 | T = 每60秒轮询一次 |
| 回滚代价 C | 漂移警告 | C_rollback = Σ\|Δbias\|·cost |
| 权重 W(x) | 整体健康分 | W = 加权均值(node_scores) |
| 关联性 R(x,y) | 节点间漂移传播 | R = 相关系数矩阵 |

### 持续性自检协议（卷83·消三虫诀映射 + 三不去原则）

```python
class HealthDashboard:
    """庚申审计引擎 — 持续性元认知层
    
    六论映射:
      元认知论（第六论）→ 系统自照（卷94「坐忘论·真观」: 智观智，心观心）
      未来观论（第五论）→ 趋势追踪（卷15「阴符经」: 观天之道，执天之行）
    
    解语师能力阶段:
      语义感知→语义推理→语义解构→语义重构→语义创生
      仪表盘当前阶段: 语义重构（能对系统状态进行结构化解析与重建）
    """
    
    def soma_gen8_audit(self) -> dict:
        """艮八: 元认知审计 — 系统自照（卷94「坐忘论·真观」）"""
        warnings = self.quality_tracker.get_drift_warnings()
        node_health = self.quality_tracker.get_node_health()
        
        # 三不去原则检查（卷81-83「庚申部」·候三尸法映射）
        assert self.test_pass_rate >= self.last_pass_rate, \
            "一不去: 验证不得退化（卷82「守庚申」不得使三尸得申）"
        assert self.bridge_count >= self.last_bridge_count, \
            "二不去: 桥不得断裂（卷5「真系谱牒」师承不断）"
        assert self.panel_count >= self.last_panel_count, \
            "三不去: 面板不得消失（卷27「洞天福地」可见性不减）"
        
        return {
            "omega": self._compute_continuity_index(),   # Ω(t) ≥ 1.0
            "soma_position": "艮八·元认知",
            "three_worm_cleared": len(warnings) == 0,
            "tuibei_health_gua": self._map_health_to_gua(),  # 推背图卦象映射
        }
    
    def _compute_continuity_index(self) -> float:
        """持续性指数完整公式（v16.0统一版）
        
        Ω(t) = (P_pass/P_total)             ← 卷117-122 灵验通过率
             × (B_t/max(B_history, 1))      ← 卷5「真系」桥增长率
             × (V_visible/V_total)          ← 卷27「洞天」面板覆盖
             × K_kinship                    ← 亲缘性K（同构度）
        
        目标: Ω(t) ≥ 1.0，∀t ∈ [0, ∞)
        """
        p_pass_rate = self.test_pass_rate / max(self.test_total, 1)
        bridge_rate = self.bridge_count / max(self.max_bridge_history, 1)
        panel_rate = self.panel_visible / max(self.panel_total, 1)
        kinship = self.k_kinship  # 当前值: 0.985（v15.0互照后）
        return p_pass_rate * bridge_rate * panel_rate * kinship
```
