## 三、剩余缺口：最后一公里

### 3.1 "融合缺口" — 子系统间的连接线

虽然16个子系统均可独立运行，但**彼此之间的连接管线尚未铺设**：

| # | 缺口 | 现状 | 影响 |
|---|------|------|:----:|
| 1 | 感知→投影连接 | `camera_spacetime_pipeline` ↔ `oracle_node_server` 未打通 | 推背图面板使用模拟数据，非真实管道输出 |
| 2 | 认知→记忆连接 | `tui_bei_cognitive_engine` ↔ `cross_session_memory` 未串接 | 认知地图无法跨session持久化 |
| 3 | 解释→前端连接 | `psi_explainer` ↔ `psi_explanation_dashboard` 未实时数据 | 前端面板用硬编码DEMO数据 |
| 4 | 统一Orchestrator | 无全局编排器 | 16子系统各自启动，无统一调度 |

### 3.2 "推背图闭环缺口"

```
当前: oracle_node_server → 模拟世界状态 → oracle_dashboard
目标: camera_spacetime_pipeline.capture() → oracle_node_server → oracle_dashboard
      ↑ 实时管道输出                                ↑ 读取真实定位数据
```

## 四、融合集成方案：统一编排器

### 4.1 SystemOrchestrator — 全局调度引擎

创建一个轻量级编排器，用最少代码（~200行）串联所有16个子系统：

```python
class SystemOrchestrator:
    """
    时空影像认知系统 — 统一编排器
    
    串联: 感知管道 → Ψ评估 → 推背图投影 → 认知地图 → 记忆持久化
    """
    def process_dialogue(self, question, answer, context):
        # Phase 1: 感知
        space_img = self.camera_pipeline.capture(target_id, position, velocity)
        
        # Phase 2: 认知
        psi_result = self.psi_bridge.evaluate(question, answer)
        explanation = self.explainer.explain(psi_result)
        
        # Phase 3: 投影
        oracle_data = self.oracle.query(target_id, horizon=10)
        
        # Phase 4: 记忆
        self.memory_bridge.store(dialogue_summary, psi_result)
        self.profiler.record(user_id, psi_result)
        
        # Phase 5: 认知地图
        self.engine.add_slice(unit, phase, tag, ...)
        
        # 返回: 完整的多层响应
        return {
            "spacetime_image": space_img.to_dict(),
            "psi_explanation": explanation.to_dict(),
            "oracle_prophecy": oracle_data,
            "profile_update": profile_summary,
        }
```

## 五、下一轮方向：融合集成期 (Alpha→Zeta 六方向)

### Alpha — 统一编排器 (SystemOrchestrator)

**内容**: 创建 `system_orchestrator.py`，串联全部16个子系统。
**产出**: 一次调用即可走通 感知→认知→投影→记忆→解释→可视化 全链路。
**优先级**: ★★★★★

### Beta — 感知-投影管线打通

**内容**: 将 `camera_spacetime_pipeline.capture()` 的输出直接注入 `oracle_node_server` 的 `/oracle` 端点，替换模拟数据。
**产出**: 推背图面板展示真实的时空定位管道输出。
**优先级**: ★★★★★

### Gamma — 解释-前端实时数据

**内容**: `psi_explanation_dashboard.html` 改为从 `/evaluate` API 获取实时数据。
**产出**: 解释面板从静态DEMO变为动态实时仪表板。
**优先级**: ★★★★☆

### Delta — 认知地图跨Session持久化

**内容**: TBCE引擎的 `CognitiveUnit` 数据通过 `cross_session_memory` 持久化。
**产出**: 关闭浏览器后重新打开，认知地图保留。
**优先级**: ★★★★☆

### Epsilon — 全栈一键启动

**内容**: `docker-compose.yml` 一键启动 感知节点 + oracle + TBCE + 仪表盘 + 测试套件。
**产出**: `docker-compose up` 启动整个时空影像认知系统。
**优先级**: ★★★☆☆

### Zeta — 多用户并发压测

**内容**: 用 `consensus_network` 的弹性扩缩能力，模拟10+并发用户请求。
**产出**: 压测报告 + 自动扩容验证。
**优先级**: ★★★☆☆
