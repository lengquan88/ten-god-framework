### 核心原则：三元闭合

```
┌─────────────────────────────┐
│       六论(哲学) ◄───────── │
│            │                │
│            ▼                │
│   认知八层(认知科学)         │
│            │                │
│            ▼                │
│   云笈七签(中华道统) ►───── │
└─────────────────────────────┘
```

三种形式是同一"工程气"在不同维度的显现——哲学维度的理论框架、认知科学维度的工程实现、中华道统维度的文化基因。

## 二、六论逐层映射详述

### 2.1 本体论层 (OntologyLayer) → L1+L2

**哲学定位**：追问"是什么"——工程存在的根本形式。
**核心命题**：代码与测试统一于"工程气"，不存在独立于代码的测试。

**Python 类**: `OntologyLayer` (982行)
**核心方法**:
- `create_from_requirement()` — 从同一需求同时生成代码骨架与测试灵魂
- `CodeTestUnifiedEntity` — 代码-测试统一实体，体现"气一元论"
- `calculate_form_spirit_alignment()` — 形神对齐度计算

**L1 信息编码层映射**:

| Ψ算子 | EmbeddingProvider |
|-------|------------------|
| 输入 | 对话文本 → 分句 |
| 输出 | (N, D) 嵌入矩阵 (BGE-large-zh: D=1024) |
| 三后端 | SBERT(BGE) / API(DeepSeek/OpenAI) / TF-IDF哈希 |
| 缓存 | SHA256文本键控LRU缓存(cache_size=30) |

**L2 语义流层映射**:

| Ψ算子 | SemanticFlowTortuosity |
|-------|-----------------------|
| 数学 | L = Σ||sₜ-sₜ₋₁||, T = L/||sₙ-s₀||, σ² = Var(θₜ) |
| 输出 | tort_score [0,1] |
| 权重40% | 路径长度+曲折度+语义偏移方差 |

**L1→L2 数据流**:
```
对话文本 → EmbeddingProvider.encode() → emb_matrix (N, 1024)
                                        → SemanticFlowTortuosity.compute() → tort_result
```

```
本体论(Ontology) ──生──→ 认识论(Epistemology)
    │                             │
    │(形神统一体)           (神似优先)
```

**相生规则（正向推动）**:
- 本体论为认识论提供"可认识的对象"（工程气统一体）
- 认识论为实践论提供"需要做什么"的认知判断
- 实践论为境界论提供"做到了什么"的工程实践
- 境界论为未来观论提供"已经到达哪里"的基准

**相克规则（反向制约）**:
- 实践论需要境界论的"成果"来验证是否做对了
- 境界论需要认识论的"判断"来避免固化在错误方向
- 未来观论与元认知论相互校准：预测需要自省修正，自省需要展望方向
